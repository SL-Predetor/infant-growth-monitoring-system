ROUTER FILES DETAILS:
---------------------

File: routers/cry_router_audio.py
- Endpoint: POST /predict-cry
- Accepts: WAV, MP3, M4A audio files
- Processing Steps:
  1. Load audio file using librosa
  2. Extract MFCC features (13-40 coefficients)
  3. Calculate mean and std of features
  4. Load scaler_a.pkl and scaler_b.pkl
  5. Scale features
  6. Run through model_a_pain.pkl
  7. Run through model_b_hunger.pkl
  8. Compare confidence scores
  9. Return highest confidence prediction
- Returns: Cry reason (Hunger/Pain/Normal) + confidence + message

File: routers/cry_router_img.py
- Endpoint: POST /predict-face
- Accepts: JPG, PNG image files
- Processing Steps:
  1. Load image using OpenCV or PIL
  2. Resize to (224, 224)
  3. Normalize pixel values
  4. Load facial_pain_model.keras
  5. Run prediction
  6. Extract class probabilities
  7. Determine Pain vs No-Pain
  8. Calculate confidence
- Returns: Pain expression + confidence + pain_probability

File: routers/cry_router_fusion.py
- Endpoint: POST /fusion/predict & POST /predict
- Accepts: JSON payload with audio/image predictions + context
- Processing Steps:
  1. Validate input data
  2. Create feature vector from all inputs
  3. Apply fusion algorithm:
     - Weight audio prediction (40%)
     - Weight image prediction (30%)
     - Weight contextual factors (30%)
  4. Determine final cry reason
  5. Calculate confidence level (Low/Medium/High)
  6. Generate context-based insights
  7. Check for model disagreement
  8. Generate recommendations
- Returns: Final prediction with comprehensive analysis

API INTEGRATION POINTS:
-----------------------

In smart-cry-analysis.tsx:

1. uploadAudio()
   Location: During Step 3 → Step 4 transition
   Calls: POST /predict-cry
   Input: audioUri (native) or audioBlob (web)
   Output: { label, confidence, message }
   Mapping: "pain_cry" → "Pain", "hunger_cry" → "Hunger"

2. uploadFace()
   Location: During Step 3 → Step 4 transition
   Calls: POST /predict-face
   Input: faceUri (image path)
   Output: { label, confidence, message }
   Mapping: "pain_expression" → "Pain", "no_pain" → "No-Pain"

3. submitAnalysis()
   Location: Step 3 Form Submission
   Calls: POST /fusion/predict
   Input: {
     audio_predicted_class: "Pain"/"Hunger"/"Normal"/"Unknown",
     audio_confidence: 0.0-1.0,
     image_predicted_class: "Pain"/"No-Pain"/"Unknown",
     image_confidence: 0.0-1.0,
     ... context data ...
   }
   Output: Fusion result with all details
   Fallback: Tries POST /predict if /fusion/predict returns 404

6. DATABASE/STORAGE REQUIREMENTS
================================================================================

CURRENT STATE:
--------------
Status: Stateless API (no database)
Advantage: Easy to scale
Limitation: Cannot persist analysis history

FOR AZURE ENHANCEMENT (Future):
------------------------------

Option 1: Azure Cosmos DB (Recommended for scalability)
  Purpose: Store analysis history
  Schema:
  {
    "id": "uuid",
    "userId": "uuid",
    "analysisDate": "ISO-8601 timestamp",
    "babyAge": number,
    "prediction": "string",
    "confidence": number,
    "audioUri": "string (optional)",
    "imageUri": "string (optional)",
    "context": {
      "feedingTime": number,
      "sleepTime": number,
      "diaperStatus": "string",
      "temperature": number
    }
  }

Option 2: Azure SQL Database
  Purpose: Structured data storage
  Tables:
  - users (id, name, email, createdAt)
  - analyses (id, userId, prediction, confidence, timestamp)
  - baby_profiles (id, userId, age, birthDate)

Option 3: Azure Blob Storage
  Purpose: Store audio/image uploads
  Structure:
  /audio/{userId}/{timestamp}.wav
  /images/{userId}/{timestamp}.jpg

Option 4: Azure Table Storage
  Purpose: Store real-time session data
  Tables:
  - UserSessions (userId, sessionId, expiresAt)
  - AnalysisCache (analysisId, result, timestamp)

IMPLEMENTATION NOT REQUIRED FOR MVP:
Backend works without database. Can be added later when needed.

8. AZURE DEPLOYMENT SCRIPT
================================================================================

COMPLETE DEPLOYMENT SCRIPT:
----------------------------

Save as: deploy.sh (for Mac/Linux) or deploy.ps1 (for Windows PowerShell)

Windows PowerShell Version (deploy.ps1):
```powershell
# Azure Deployment Script for Infant Cry Analysis Backend
# Date: February 2026
# Instructions: Run this script to deploy to Azure

Write-Host "🚀 Starting Azure Deployment..." -ForegroundColor Green

# Variables
$resourceGroup = "infant-cry-rg"
$location = "eastus"
$appServicePlan = "infant-cry-plan"
$appName = "infant-cry-backend"
$pythonVersion = "3.9"

# Step 1: Login to Azure
Write-Host "Step 1: Logging into Azure..." -ForegroundColor Yellow
az login

# Step 2: Create Resource Group
Write-Host "Step 2: Creating Resource Group..." -ForegroundColor Yellow
az group create `
  --name $resourceGroup `
  --location $location

# Step 3: Create App Service Plan
Write-Host "Step 3: Creating App Service Plan..." -ForegroundColor Yellow
az appservice plan create `
  --name $appServicePlan `
  --resource-group $resourceGroup `
  --sku B1 `
  --is-linux

# Step 4: Create Web App
Write-Host "Step 4: Creating Web App..." -ForegroundColor Yellow
az webapp create `
  --resource-group $resourceGroup `
  --plan $appServicePlan `
  --name $appName `
  --runtime "python|$pythonVersion"

# Step 5: Configure App Settings
Write-Host "Step 5: Configuring App Settings..." -ForegroundColor Yellow
az webapp config appsettings set `
  --resource-group $resourceGroup `
  --name $appName `
  --settings `
    WEBSITES_PORT=8000 `
    MODEL_PATH=/app/mlModels/Cry/ `
    ENVIRONMENT=production `
    LOG_LEVEL=INFO

# Step 6: Set Startup Command
Write-Host "Step 6: Setting Startup Command..." -ForegroundColor Yellow
az webapp config set `
  --resource-group $resourceGroup `
  --name $appName `
  --startup-file "python -m uvicorn app:app --host 0.0.0.0 --port 8000"

# Step 7: Enable Local Git Deployment
Write-Host "Step 7: Enabling Local Git Deployment..." -ForegroundColor Yellow
$deploymentUrl = az webapp deployment source config-local-git `
  --resource-group $resourceGroup `
  --name $appName `
  --query url `
  --output tsv

Write-Host "Git Remote URL: $deploymentUrl" -ForegroundColor Green

# Step 8: Deploy Code
Write-Host "Step 8: Deploying Code..." -ForegroundColor Yellow
cd backEnd
git init
git config user.email "deployment@example.com"
git config user.name "Azure Deployment"
git add .
git commit -m "Initial backend deployment"
git remote add azure $deploymentUrl
git push azure master

# Step 9: Verify Deployment
Write-Host "Step 9: Verifying Deployment..." -ForegroundColor Yellow
Write-Host "Waiting for deployment to complete (1-2 minutes)..." -ForegroundColor Cyan
Start-Sleep -Seconds 30

$statusUrl = "https://$appName.azurewebsites.net/docs"
Write-Host "Testing API endpoint: $statusUrl" -ForegroundColor Yellow

# Wait for service to be ready
$maxAttempts = 10
$attempt = 0
while ($attempt -lt $maxAttempts) {
    try {
        $response = Invoke-WebRequest -Uri $statusUrl -ErrorAction Stop
        Write-Host "✅ API is responding!" -ForegroundColor Green
        break
    }
    catch {
        $attempt++
        if ($attempt -lt $maxAttempts) {
            Write-Host "⏳ Waiting... (Attempt $attempt/$maxAttempts)" -ForegroundColor Yellow
            Start-Sleep -Seconds 10
        }
    }
}

# Step 10: Display Deployment Summary
Write-Host "`n" -ForegroundColor Green
Write-Host "================================" -ForegroundColor Green
Write-Host "✅ DEPLOYMENT SUCCESSFUL!" -ForegroundColor Green
Write-Host "================================" -ForegroundColor Green
Write-Host "Service URL: https://$appName.azurewebsites.net" -ForegroundColor Green
Write-Host "API Docs: https://$appName.azurewebsites.net/docs" -ForegroundColor Green
Write-Host "Swagger UI: https://$appName.azurewebsites.net/redoc" -ForegroundColor Green
Write-Host "Resource Group: $resourceGroup" -ForegroundColor Green
Write-Host "================================" -ForegroundColor Green

# Step 11: Show Logs
Write-Host "`nTo view deployment logs, run:" -ForegroundColor Yellow
Write-Host "az webapp log tail --resource-group $resourceGroup --name $appName" -ForegroundColor Cyan
```
