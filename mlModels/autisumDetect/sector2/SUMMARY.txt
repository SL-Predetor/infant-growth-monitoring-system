================================================================================
SECTOR 2 — Q-CHAT-10 Questionnaire-Based ASD Detection
Summary of Notebooks
================================================================================

OVERVIEW
--------
Sector 2 implements a questionnaire-based ASD detection pipeline using the
Q-CHAT-10 screening tool. The pipeline has three layers:

  Layer 1 (Q-CHAT-10/) : Individual dataset preprocessing and model training
                          on single-source data (toddlers / children).

  Layer 2 (Stage_0/)   : Multi-source data fusion — combines 5 compatible
                          Q-CHAT-10 datasets into one large unified dataset
                          for more robust downstream modelling.

  Layer 3 (Stage_1/, Stage_2/) : Feature ablation and population filtering —
                          Stage_1 removes administrative bias features;
                          Stage_2 further filters to parent/family-reported
                          rows only (deployment-realistic population).

1. Q01.ipynb — Data Preprocessing (Toddlers Dataset)
   --------------------------------------------------
   Dataset : "Autism Dataset for Toddlers.csv"
   Initial rows : 1054 → 975 after removing 79 duplicates
   No missing values found.

   Features:
     - A1–A10     : Binary Q-CHAT-10 questionnaire responses (0/1)
     - Age_Mons   : Age in months (12–36 months range)
     - Qchat-10-Score : Total score (0–10)
     - Sex, Ethnicity, Jaundice, Family_mem_with_ASD,
       Who completed the test
   Target : Class/ASD Traits (Yes/No)

   Preprocessing steps:
     - Dropped Case_No and duplicate rows
     - No outliers found in Age_Mons or Qchat-10-Score (IQR method)
     - Cleaned "Who completed the test": merged 'Health care professional'
       → 'Health Care Professional', 'Self' → 'Others'
     - Dropped all Ethnicity_* columns (deemed non-informative)
     - One-hot encoded: Sex, Jaundice, Family_mem_with_ASD,
       Who completed the test, Class/ASD Traits
     - StandardScaler applied to Age_Mons and Qchat-10-Score
   Output : pre_processed_data.csv

5. Stage_0/combine_datasets.ipynb — Multi-Source Dataset Fusion
   -------------------------------------------------------------
   Purpose : Aggregate all compatible Q-CHAT-10 datasets into one large
             unified preprocessed dataset for more robust downstream modelling.

   Sources included:
     Dataset                              | Raw Rows | After Dedup
     -------------------------------------|----------|-----------
     Autism Dataset for Toddlers.csv      |    1054  |       920
     Children_ASD.csv                     |    2226  |      1105
     Autism-Child-Data.arff               |     288* |       272
     Autism-Adolescent-Data.arff          |     104  |       103
     Autism_Screening_Data_Combined.csv   |    6075  |      5130
     TOTAL                                |    9747  |      7530

     * 4 rows dropped due to missing age values

   Sources excluded:
     - "Toddler Autism dataset July 2018.csv" — verified exact duplicate
       of Toddlers CSV
     - "data1.csv" — incompatible schema (25-item QCHAT, not 10-item)

   Common raw schema (normalised before concatenation):
     A1–A10, Age_in_Months, Sex, Jaundice, Family_mem_with_ASD,
     Who_completed_the_test, ASD_traits, source

   Age coverage after combining:
     - Toddlers      : 12–36 months
     - Children      : up to 120 months (10 years)
     - Adolescents   : up to 192 months (16 years)

   Normalisation helpers used:
     - normalise_who()   : unifies 'parent'/'relative' → 'Family Member', etc.
     - normalise_yesno() : standardises yes/no across datasets
     - normalise_sex()   : standardises M/F casing
     - normalise_asd()   : standardises YES/NO/y/n → 'Yes'/'No'

   Deduplication:
     - Based on all feature columns (source tag excluded)
     - keep='first' strategy → 2217 cross-dataset duplicates removed

   Preprocessing (to match QM1.ipynb schema):
     - Dropped  : source, Age_in_Months, Jaundice
     - Renamed  : A10 → A10_Autism_Spectrum_Quotient (retained as A10 in combined)
     - Encoded  : OHE on Sex, Family_mem_with_ASD,
                  Who_completed_the_test, ASD_traits (drop_first=True)
     - No StandardScaler (Age_in_Months dropped; no continuous features remain)

   Final output shape : 7530 rows × 16 columns
   Output file        : pre_processed_data_combined.csv

   Final encoded features (16 total):
     A1–A9, A10_Autism_Spectrum_Quotient, Sex_M, Family_mem_with_ASD_Yes,
     Who_completed_the_test_Health Care Professional,
     Who_completed_the_test_Others,
     Who_completed_the_test_School and NGO,
     ASD_traits_Yes (target)

   Class balance (after dedup):
     Non-ASD (False) : 4414  (58.6%)
     ASD     (True)  : 3116  (41.4%)
     → Moderately imbalanced; manageable without heavy resampling
