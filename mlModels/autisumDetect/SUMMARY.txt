PROJECT OVERVIEW:
A multi-stage machine learning system to detect Autism Spectrum Disorder (ASD)
in infants using TWO parallel approaches:
  1. FACIAL IMAGE ANALYSIS  - Modified VGG-Face transfer learning on infant photos
  2. QUESTIONNAIRE ANALYSIS - Machine learning on Q-CHAT-10 survey responses

DATASET (Facial):
  - Total images: 3,917  (1,952 Autistic | 1,965 Non-Autistic)
  - Source: step4/ directory (preprocessed, deduplicated, augmented)
  - Blind test set (permanent, isolated before any training): 392 images
      Autistic=195, Non_Autistic=197
  - Label convention enforced in code: Autistic = 1, Non_Autistic = 0
    (alphabetical order gives Autistic index 0, so labels are flipped via 1 - y)

===== PREP FOLDER - DATA PREPARATION & PREPROCESSING =====

1. prep1.ipynb
   ├─ PURPOSE: Face Detection & Facial Region Cropping
   ├─ LIBRARY: MTCNN (Multi-task Cascaded Convolutional Networks)
   ├─ PROCESS:
   │   • Reads raw infant photos from: rawimg/Non_Autistic (and Autistic)
   │   • Detects faces using MTCNN detector
   │   • Crops detected face with 10% (margin=0.1) padding on all sides
   │   • Saves cropped faces as sequential JPEGs to prepImg/step1/
   │   • Deletes original raw file after successful processing
   ├─ INPUT: rawimg/{Class}/  (raw infant photographs, various sizes/formats)
   ├─ OUTPUT: prepImg/step1/{Class}/  (cropped face images)
   └─ NOTE: An alternative DeepFace + RetinaFace approach is present as
             commented-out code but was not used in the main pipeline.

2. prep2.ipynb
   ├─ PURPOSE: Duplicate Image Detection & Removal
   ├─ METHOD: MD5 Cryptographic Hashing
   ├─ PROCESS:
   │   • Reads from prepImg/step3(S1+S2.2)/ (combined step1 + step2 images)
   │   • Computes MD5 hash for every image
   │   • Identifies duplicates by matching hash values
   │   • Moves only unique originals to prepImg/step4/
   ├─ ACTUAL RESULT: Found 29 duplicates in the Non_Autistic class
   ├─ INPUT: prepImg/step3(S1+S2.2)/{Class}/
   ├─ OUTPUT: prepImg/step4/{Class}/  (final clean dataset used for training)
   └─ IMPORTANCE: Prevents data leakage across train/test splits.

3. augment1.ipynb
   ├─ PURPOSE: Data Augmentation (Synthetic Sample Generation)
   ├─ INPUT: rawimg/{Class}/  (applied on raw images, not cropped faces)
   ├─ OUTPUT: prepImg/step2/{Class}/  (augmented images, prefix "aug_")
   ├─ SELECTION: Randomly selects 500 images from input for augmentation
   ├─ AUGMENTATION TECHNIQUES (6 total, max 300 applications each):
   │   • Random Rotation:    ±30 degrees
   │   • Horizontal Flip:    Mirror left-to-right
   │   • Zoom:               1.1x to 1.5x magnification with centre crop
   │   • Positional Shift:   ±30 pixel random translation (affine transform)
   │   • Brightness:         0.7x to 1.3x scaling via ImageEnhance
   │   • Contrast:           0.7x to 1.3x scaling via ImageEnhance
   ├─ LOGGING: Progress logged to process.log via Python logging module
   └─ NOTE: step2 output is later merged with step1 into step3, which is then
             deduplicated by prep2.ipynb to produce the final step4 dataset.

4. changeName.ipynb
   ├─ PURPOSE: File Renaming & Standardized Organization
   ├─ PROCESS:
   │   • Reads from prepImg/step2/{Class}/ (augmented images)
   │   • Renames all files to sequential integers: 0.jpg, 1.jpg, 2.jpg, ...
   │   • Moves renamed files to a new output folder
   ├─ INPUT: prepImg/step2/{Class}/
   ├─ OUTPUT: prepImg/New folder/{Class}/  (standardized naming)
   └─ IMPORTANCE: Creates consistent filenames for reproducible dataset loading.

===== STAGE_1 FOLDER - BASELINE MODEL TRAINING =====

SHARED ARCHITECTURE (model1.ipynb through model5.ipynb):
  All Stage_1 models use an identical dual-output VGG-Face architecture:

  INPUT (224×224×3)
    → VGG-Face Backbone [conv2d_14 output, frozen] → (None, 1, 1, 4096)
    → Global Average Pooling → (None, 4096)
    → Dense(256, relu) [named: asd_feature_vector]
    → Dense(1, sigmoid) [named: classification_output]

  Total params:     135,309,633
  Trainable (head): 1,049,089
  Frozen (backbone):134,260,544

TRAINING PROTOCOL (shared):
  • Phase 1 - Frozen Backbone:
      LR=0.0001, Adam optimizer, binary_crossentropy loss
      20 epochs + optional 15-epoch extension (35 total)
      EarlyStopping (patience=5 on val_loss)
      ModelCheckpoint (best val_accuracy)
  • Phase 2 - Partial Fine-Tuning:
      Unfreeze last 4 layers of VGG-Face backbone
      LR=1e-5, batch_size=16, up to 10 epochs
      EarlyStopping (patience=5); early stopping triggered in all models
  • Data split: 80% train (3134) / 10% val / 10% test (from 3917 images)
  • Preprocessing: Pixel normalization to [0,1] via Rescaling(1./255)
    NOTE: Stage_1 uses simple normalization. Stage_2+ uses proper VGG-Face
    preprocessing (BGR + mean subtraction) - see ML1.ipynb and Stage_4.

===== STAGE_4 FOLDER - IMPROVED K-FOLD WITH MIXED PRECISION =====

MOTIVATION:
  Stage_4 addresses two problems from Stage_2:
    1. OOM errors during Phase 2 fine-tuning on 6GB VRAM GPU (conv2d_13
       gradient tensor ~400MB under mixed precision)
    2. Limited head capacity (single Dense(256)) and no BatchNorm

ARCHITECTURE (improved head, same VGG-Face backbone cut point):
  INPUT (224×224×3)
    → VGG-Face Backbone [conv2d_14 output, frozen in Phase 1]
    → GlobalAveragePooling2D
    → Dense(512, relu, L2=0.001)  [named: asd_feature_vector_512]
    → BatchNormalization
    → Dropout(0.5)
    → Dense(256, relu, L2=0.001)  [named: asd_feature_vector_256]
    → Dropout(0.4)
    → Dense(1, sigmoid, dtype=float32)  [named: classification_output]

  Trainable params (head only, Phase 1): ~2,230,273
  Mixed precision policy: mixed_float16 (output layer forced float32)

KEY IMPROVEMENTS OVER STAGE_2:
  • Mixed precision training (float16 activations, float32 accumulation)
  • set_memory_growth(True) prevents TF pre-allocating all 6GB VRAM
  • UNFREEZE_LAYERS=2 in Phase 2 (only conv2d_14 + activation)
    — avoids conv2d_13 (7×7×512→4096, ~400MB gradient under fp16)
  • LR_PHASE2=1e-6 (conservative, suitable for actual backbone weights)
  • prefetch(1) instead of AUTOTUNE (caps dataset buffering on 6GB GPU)
  • Full cleanup per fold: del all model/backbone/dataset/history references
  • Two-session split: Phase 1 and Phase 2 run in separate scripts/notebooks
    with a kernel restart in between (fully resets CUDA VRAM context)
  • class_weight = {0: 1.0, 1: 2.0} — Autistic class gets 2× penalty
  • StratifiedKFold (same 5-fold splits as Stage_2 reloaded from .npy files)
