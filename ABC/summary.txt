2. Does growth.tsx fetch from the backend API?
Yes, it fetches from the backend. It calls:

GET http://localhost:8000/api/growth/dashboard/${infantData.id}
It also makes a second direct Supabase call to check if today's daily log exists.

3. Where does measurements/chart data come from?
From the growth dashboard API response. The field is data.chart_data which is set into measurements state. The backend fetches from the Supabase measurements table server-side and returns it inside the dashboard response. The frontend never queries measurements directly.

8. How is infant_id obtained?
Fetched fresh from Supabase each time — not stored in context or AsyncStorage. The pattern used throughout the app:

const { user } = useAuth();  // gets Supabase auth user (has user.id)

// Then inside useEffect:
const { data: infantData } = await supabase
  .from('infants')
  .select('*')
  .eq('parent_id', user.id)
  .maybeSingle();

// infantData.id is the infant_id used for API calls

user.id comes from AuthContext which wraps Supabase session.user. infant_id is the Supabase UUID of the row in the infants table where parent_id = user.id. There is one infant per parent (.maybeSingle()).
